Contact pointop back to ToC or Object Property ToC

IRI: https://w3id.org/nap/model/pointOfContact

Describes an organisation, if applicable a person, which is responsible for creation and maintenance of the NAP itself. This person or company is the direct contact for NAP users, who have questions or issues about the NAP system, for example. This information is mandatory and should correspond to the information in the disclaimer of the NAP website.
has domain
NAP entry c or NAP Metadata c
has range
contact point c

Data ownerop back to ToC or Object Property ToC

IRI: https://w3id.org/nap/model/dataOwner

Describes the entity that legally owns the data provided in a dataset or service. This entity is responsible for the content of the data, in particular, that the data fulfilles quality criteria (if applicable, see “Quality description” below) and/or corresponds to legal frameworks (if applicable, see “Legal frameworks” below). The “Data Owner” will be in many cases identical with the “Publisher”. In this case, the contact data will be copied from the “Publisher” entry. There might be also cases where a “Data Owner” contracts a third-party “Publisher” (e.g., an IT service company) to host and publish a data set. This entity is the direct contact for the NAP operator or data-consuming NAP users, who have questions or issues about the contents of a dataset, for example. This information is mandatory, and should include, as a minimum, the name and email address of the entity.
has domain
Data set or service c
has range
creator c

Dataset publisherop back to ToC or Object Property ToC

IRI: https://w3id.org/nap/model/publisher

Describes an entity (company and person) that publishes the corresponding data set or service. This entity is responsible for the provisioning of a dataset or service. The entity also concludes a contract, if applicable. This entity is the direct contact for the NAP operator or data-consuming NAP users, who have questions or issues about the availability of a dataset, for example. This information is mandatory, and should include, as a minimum, the name and email address of the entity. The Publisher might be identical to the “Data Owner” (see below).
has domain
Data set or service c
has range
publisher c

Dataset visibilityop back to ToC or Object Property ToC

IRI: https://w3id.org/nap/model/visibility

Describes the visibility status of a data set or data service publication on the NAP interface. It is used only for internal usage or administration via predefined keys such as “public” or “private”. “Public” means that the dataset or service can be retrieved by external NAP users, or that that the corresponding metadata can be exchanged. “Private” means that the dataset, the service and the metadata are only visible to the original data provider and the NAP operator; this might be the case for test publications, for example.
has domain
Data set or service c
has range
visibility c

Languageop back to ToC or Object Property ToC

IRI: https://w3id.org/nap/model/language

This element indicates the language in which the metadata is described. Next to the national operators and publishers there are international operators which use their own language for descriptions. According to the ISO 639 standard part 2, there is a 3 letter code for 24 EU languages, which should be used. At least one language has to be set, but it should be possible to select more than one language.
has domain
NAP entry c
has range
language c
